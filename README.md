# Linux System Administration and Security Hardening

## Overview

This repository contains a sanitised Linux system administration and security hardening project completed in a controlled academic lab environment.

The project focuses on practical Linux service management, Apache HTTP Server administration, MariaDB database service management, firewall configuration, vulnerability analysis, risk assessment, and hardening of default server configurations.

## Project Objectives

- Manage Linux services using `systemd`
- Install, configure, and verify Apache HTTP Server
- Manage MariaDB service and database user privileges
- Configure Linux firewall rules using `firewalld`, `iptables`, and UFW
- Identify weaknesses in default Apache and firewall configurations
- Apply security hardening controls to reduce attack surface
- Verify improvements using command-line testing and before/after evidence
- Document risk reduction through structured technical analysis

## Summary

This project demonstrates how insecure or default Linux server configurations can be reviewed, assessed, hardened, and verified.

The hardening process focused on improving confidentiality, integrity, and availability by reducing information disclosure, enforcing encrypted communication, restricting unnecessary services, limiting exposed ports, and applying the principle of least privilege.
